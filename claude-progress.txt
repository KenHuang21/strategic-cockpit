TECHNICAL NOTES:
===============

Frontend Architecture:
- Next.js 14 with App Router (server components by default)
- Client components marked with "use client" directive
- API routes serve data from JSON files in ../data/ directory
- Refresh API triggers GitHub Actions via repository_dispatch
- Types defined in /lib/types.ts for type safety

Data Flow:
1. Python scripts fetch data from APIs
2. Scripts write to JSON files in /data/ directory
3. Scripts commit changes to repository
4. Next.js API routes read from /data/ directory
5. Frontend components fetch from API routes
6. Vercel rebuilds/redeploys automatically on git push

Current Limitations:
- Data files contain placeholder data (zeros)
- GitHub Actions workflows not yet created
- No actual API integrations yet
- Settings Modal not implemented
- Documentation page not created
- No notification system yet
